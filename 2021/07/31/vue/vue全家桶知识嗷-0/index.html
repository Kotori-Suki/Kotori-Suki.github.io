# 1. Vue 基础

# 1.0 MVVM 模式

  • M 模型 (Model)：对应 data 中的数据
  • V 视图 (View)：模板
  • VM 视图模型 (ViewModel)：Vue 对象实例

把一堆乱七八糟的数据（模板）和一堆 dom 结构通过数据模型做一个连接，把数据放在框架要求放好的位置，然后写出模板代码，就能建立起一个连接

而 Vue 中数据通过 Vue 实例进行数据绑定，就能把你的数据渲染到视图当中去，而通过 Vue 实例的时间监听使页面的改变也能映射到数据的改变

# 1.1 Vue 介绍

  • vue 是什么
    • 一套用于构建用户界面的渐进式的 JavaScript 框架（拿到手里的数据通过 vue 变成用户可以看到的界面）
  • 特点
    • 采用了组件化模式，提高了代码的复用率，且让代码更好维护
    • 声明式编码，让编程人员无需直接操作 DOM，提高开发效率
    • 使用虚拟 DOM + 优秀的 Diff 算法，尽量复用 DOM 节点

# 1.2 创建 Vue 实例

data 中的数据发生改变，那么页面中用到这个数据的地方也会自动更新

# 1.3 Vue 模板语法

# 1.3.1 差值语法
  • 功能：用于解析标签体内容
  • 写法： {{xxx}} ，xxx 是 js 表达式，且可以直接读取到 data 中的属性（vm 实例上面有的都可以直接用）
# 1.3.2 指令语法
  • 功能：用于解析标签（标签属性，标签体内容，绑定事件 ......）
  • 指令：
    • v-text：向其所在的节点中渲染文本内容
    • v-html：向指定节点中渲染包含 html 结构的内容（动态渲染 html 是非常危险的，容易导致 CSS 攻击，永远不要相信用户的输入）
    • v-cloak (没有值)：Vue 接管容器后会删掉该属性，配合 css 使用可以解决网速慢时页面展示 问题
    • v-once (没有值)：初次动态渲染后，就视为静态内容了，以为不会引起该指令所在的结构更新，可以用于新能优化
    • v-pre (没有值)：跳过所在节点的编译过程，可利用他跳过没有 Vue 模板语法的节点，加快编译
    • v-bind : 单向绑定，数据只能从 data 流向页面（简写为 : ）
    • v-model：数据不仅仅可以从 data 流向页面，还能从页面流向 data, 默认找 value 属性（双向绑定一般都应用在表单类元素上）
    • v-on：绑定事件，可简写成 @
    • v-if /v-show：条件渲染
    • v-for：列表渲染

# 1.4 Object.defineProperty 方法

给一个对象定义属性用的，三个参数（指定对象，需要添加的参数，配置项）

配置项：

  • value ：添加属性的值
  • enumerable：控制属性是否可以枚举，默认值是 false
  • writable：控制属性是否可以被修改，默认值是 false
  • configurable：控制属性是否可以被删除，默认值是 false
  • getter：即 get 函数，当有人读取对象的属性值时，get 函数会被调用，且返回值就是属性的值
  • setter：即 set 函数，当有人修改对象的属性值时，set 函数会被调用，且返回值是修改的值

# 1.5 理解数据代理

数据代理：通过一个对象代理对另一个对象中属性的操作（读 / 写）

  • 原理：通过 Object.defineProperty 的 setter 来让另一个对象的属性值等于这个对象的 value，当这个对象的 value 被修改时，就会通过 setter 让另一个对象的值也一起改变

vue 中的数据代理

  • 是指通过 vm 对象来代理 data 对象中属性的操作（读 / 写）
  • 优点：更加方便的操作 data 中的数据（如果不代理，我们在模板语法中就要使用_data.xxx 来读取属性值）
  • 原理：通过 Object.defineProperty（）把 data 对象中的所有属性添加到 vm 上，为每一个添加到 vm 上的属性，都指定一个 getter/setter，在 getter/setter 内部去操作（读 / 写）data 中对应的属性

# 1.6 事件处理

# 1.6.1 事件的基本使用
  • 使用 v-on:xxx 或 @xxx 来绑定事件，其中 xxx 是事件名
  • 事件的回调需要配置在 methods 对象中，最终会在 vm 上
  • methods 中配置的函数，不要用箭头函数！否则里面的 this 就不是 vm 了（是 vue 实例执行时的上下文）
  • methods 中配置的函数，都是 Vue 所管理的函数，this 的指向是 vm 或组件实例对象
  • 不传参直接写 @click = ‘demo’即可，需要传参则要写 @click = ‘demo (a,b)’，需要用 event 需要在参数列表中写 $event 占位符
# 1.6.2 事件修饰符（可以连写）
  • 阻止事件冒泡
    • @click.stop = "xxx"
    • es5 方法 e.stopPropatation ()
  • 阻止事件默认行为
    • @click.prevent = "xxx"
    • es5 方法 e.preventDefault ()
  • 事件只触发一次：@click.once = "xxx"
  • 使用事件的捕获模式：@click.capture = "xxx"
  • 只有 event.target 是当前操作的元素时才触发事件：@click.self = "xxx"
  • 事件的默认行为立即执行：@click.passive = "xxx"
# 1.6.3 键盘事件

用法：@keyup.enter = "xxx"

  • Vue 中常用的按键别名：
    • 回车 => enter
    • 删除 => delete（捕获” 删除 “和” 退格 “键）
    • 退出 => esc
    • 空格 => space
    • 换行 => tab （配合 keydown 使用，keyup 的话在触发事件的时候会触发 tab 的默认行为把焦点从当前元素上切走，触发不了 keyup 事件）
    • 上下左右 => up,down,left,right
  • Vue 没有提供别名的可以根据原始的 key 值去绑定，但要注意转换为 kabab-case（短横线命名）
  • 系统修饰键（用法特殊）：ctrl，alt，shift，meta (win)
    • 配合 keyup 使用：按下修饰键的同时再按下其他键，在释放其他键，事件才会触发
    • 配合 keydown 使用：正常触发事件
    • 设置组合键：@keyup.ctrl.y => 按下组合键 ctrl 和 y 触发事件
  • 可以使用 keyCode 去指定具体的按键（不推荐，KeyboardEvent.keyCode 已经从 web 标准中移除了）
  • Vue.config.keyCodes. 自定义别名 = 键码，来指定定制按键名

# 1.7 计算属性 - computed

  • 定义：要用的属性不存在，要通过已有属性计算得来
  • 原理：底层借助了 Object.defineProperty 方法提供的 getter 和 setter

  • 优点：与 methods 实现相比，内部有缓存机制，效率更高，调试方便

  • 注意：

    • 计算属性最终会出现在 vue 实例上，直接读取就可以使用他（具有缓存功能）
    • 如果计算属性要被修改，那必须写 set 函数去响应修改，且 set 中要引起计算时依赖的数据发生改变

  • 简写形式（只有当计算属性的值不会被修改的时候，才可以简写）

# 1.8 监听属性 - watch

  • 当监视的属性变化时，回调函数自动调用，进行相关操作
  • 监视的属性必须存在，才能进行监视
  • 监视的两种写法
    • new Vue 时传入 watch 配置
    • 通过 vm.$watch 监视

  • 简写形式（当配置项只有 handler 的时候，才能写成简写模式）

# 1.8.1 深度监视
  • Vue 中的 watch 默认不检测对象内部值的改变（一层）
  • 配置 deep:true 可以检测到对象内部值的改变（多层）

  • 注意：
    • Vue 自身可以监测对象内部值的改变，但 Vue 提供的 watch 默认不可以（为了提高效率）
    • 使用 watch 时根据数据的具体结构，决定是否使用深度监视

# Ex. computed 和 watch 的区别

  • computed 能完成的 watch 也可以完成

  • watch 能完成的 computed 不一定可以完成，例如 watch 可以进行异步操作

  • 两个重要的原则：

    • 被 Vue 管理的函数，最好写成普通函数，这样 this 才能指向 vm 或者实例对象
    • 所有不被 Vue 管理的函数（定时器回调函数，ajax 回调函数，Promise 的回调函数），最好写成箭头函数，这样 this 的指向才是 vm 或 组件实例对象

# 1.10 条件渲染

# 1.10.1 v-if
  • 写法：v-if = “表达式” v-else-if = “表达式” v-else
  • 适用：切换频率较低的场景
  • 特点：不展示 DOM 元素直接被移除
# 1.10.2 v-show
  • 写法：v-show = "表达式"
  • 适用：切换频率较高的场景
  • 特点：不展示 DOM 元素未被移除，仅仅是适用样式隐藏掉（display:none）

注意：使用 v-if 的时候，元素可能无法获取到，而使用 v-show 一定可以获取到

# 1.11 列表渲染

# 1.11.1 用法

语法：v-for="item in itemList" :key="item.id" (item 是列表项，itemList 是列表，key 必须指定列表的唯一值)

  • 遍历数组

  • 遍历对象

# 1.11.2 key 的作用

  • 虚拟 DOM 中 key 的作用

    • key 是虚拟 DOM 对象的标识，当状态中的数据发生改变时，Vue 会根据新数据生成新的虚拟 DOM, 随后 Vue 进行新虚拟 DOM 和旧虚拟 DOM 的差异比较

  • 对比规则:

    • 旧虚拟 DOM 中找到了与新虚拟 DOM 相同的 key, 若虚拟 DOM 中内容没变，直接使用之前的真实 DOM, 如果发生了改变，则会生成新的真是 DOM, 随后替换掉原来的真是 DOM
    • 旧虚拟 DOM 没有找到与新虚拟 DOM 相同的 key 就会创建新的真是 DOM, 随后渲染到页面

  • 用 index 作为 key 可能会引发的问题:

    • 若数据进行：逆序添加，逆序删除等破坏顺序操作，会产生没有必要的真实 DOM 更新 => 界面效果没问题，但效率低
    • 如果结构中包含输入类的 DOM, 会产生错误的 DOM 更新 => 界面有问题

  • 开发中如何选用 key

    • 最好使用每条数据的唯一标识作为 key, 比如 id, 手机号，身份证号，学号等
    • 如果不存在对数据进行添加，逆序删除等破坏顺序操作，仅用于渲染到列表用于展示，使用 index 是没有问题的

# 1.12 Vue 监视数据原理

Vue 会监视 data 中所有层次的数据

Vue 拿到你传入的 data 之后，加工你的 data（vm._data = data，加上了数据劫持（加了 getter 和 setter）), 加工完后就可以做响应式，这时_data 里面的属性改变，就会调用 setter，重新解析模板，生成新的虚拟 DOM，然后新旧虚拟 DOM 对比，生成新的页面

  • 监测对象中的数据（无论多少层数据，他都会给你加上 setter 和 getter）
    • 通过 setter 实现监视，且要在 new Vue 时就传入要监测的数据
    • (1) . 对象中后追加的属性，Vue 默认不做响应式处理
    • (2) . 如需给后添加的属性做响应式，请使用 Vue.set ()

  • 监测数组的数据（数组的每一项没有 setter 和 getter，直接通过数组下标修改数组里面的元素不会 Vue 响应）

    • 通过包裹数组更新元素的方法实现，本质就是做了两件事:
    • (1) . 调用原生对应的方法对数组进行更新
    • (2) . 重新解析模板，进而更新页面

  • 在 Vue 修改数组中的某个元素一定要用如下方法:

    • 使用 push () 、pop ()、shift ()、unshift ()、splice ()、sort ()、reverse ()
    • Vue.set () 或 vm.$set ()

  • EX. Vue.set () 和 vm.$set () 不能给 vm 更数据对象 添加属性！

# 1.13 收集表单数据

  • <input type="text" />, 则 v-model 收集的就是 value 值，用户输入的就是 value 值
  • <input type="radio" />, 则 v-model 收集的就是 value 值，且要给标签配置 value 值
  • <input type="checkbox" />
    • 没有配置 input 的 value 属性，那么收集的就是 checked（勾选 或 未勾选，是一个 bool 值）
    • 配置了 value 属性，v-model 的初始值是非数组，那么收集的就是 checked，如果是数组，那么收集的就是 value 组成的数组
  • v-model 常用的三个修饰符
    • lazy：失去焦点时再收集数据
    • number：输入字符串转为有效的数字
    • trim：输入收尾空格过滤

# 1.14 过滤器 - filters

对要显示的数据进行特定格式化后在显示（适用于一些简单逻辑的处理）

  • 语法：
    • 注册过滤器： Vue.filter(name, callback) 或 new Vue(filters: {})
    • 使用过滤器： {{xxx | 过滤器名}} 或 v-bind:属性 = "xxx | 过滤器名"
  • 注意：
    • 过滤器也可以接受额外参数，多个过滤器也可以串联
    • 并没有改变原来的数据，是产生新的对应的数据

# EX1 cookie 的工作流程

第一次请求服务器的时候，输入表单信息请求服务器，服务器返回，登陆成功就跳个人中心，此时服务器会返回该网站的 cookie，该 cookie 会存在在用户本地磁盘里，当用户再次访问网站时，浏览器会找到 cookie，如果有服务器返回的 cookie，浏览器就会自动使用 cookie 进行自动登录，所以 cookie 的信息非常重要。

使用 v-html 是不安全的：我们使用不同浏览器是不可能访问相互的 cookie 的，一旦 cookie 泄露，别人可以在 cookie 中导入你的 cookie 即可登录你的账号，这时用户通过输入一些 html 语句且有执行 js 语法的（document.cookie），而你使用 v-html 渲染的就会暴露你的 cookie, 所以永远不要相信用户的输入

# 1.15 自定义指令 - directives

函数简写式（不能配置，处理不了一些细节的问题）

对象式 (可以配置指令绑定时，插入时，更新时的回调)

  • 定义局部指令语法： directives: {指令名: 配置对象} 或 directives: { 指令名(){} }
  • 定义全局指令语法： Vue.directive(指令名, 配置对象) 或 Vue.directive(指令名, 回调函数)

注意：

  • 指令定义时不加 v-，但是使用的时候要加 v-
  • 指令名如果有多个单词，需要使用 kebab-case 命名方式，不要用 camelCase 命名

# 1.16 生命周期函数

  • beforeCreate () => 此时 Vue 初始化了生命周期，事件，但数据代理还没开始，无法通过 vm 访问到 data 中的数据，methods 中的方法
  • created () => 初始化了数据监测，数据代理，可以通过 vm 访问到 data 中的数据，methods 中的方法
  • 看你有没有 el 的配置项或有没有调用 vm.$mount (el)，没有将报错，有就看你有没有 template 配置项，若没有就把最外层的盒子作为模板，有就使用 render 函数编译模板，这个时候 Vue 开始解析模板，生成虚拟 DOM 到内存中，页面还不能显示解析好的内容
  • beforeMount () => 此时页面呈现的时未经 Vue 编译的 DOM 结构，所有对 DOM 的操作，最终都不会奏效
  • 将内存的虚拟 DOM 转换成真实 DOM 插入页面
  • mounted () => 页面中呈现的时经过 Vue 编译的 DOM，此时对 DOM 的操作都有效，但不推荐直接操作 DOM，至此初始化过程结束，一般再次进行：开启定时器，发送网络请求，订阅消息，绑定自定义事件等初始化操作
  • 当数据发生改变时
  • beforeUpdate () => 此时数据时新的，但页面时旧的，页面尚未和数据保持同步
  • 根据新数据，生成新的虚拟 DOM，随后与旧的虚拟 DOM 进行比较，最终完成页面的更新，即完成了 Model -> View 的更新
  • updated () => 此时数据是新的，页面也是新的，页面和数据保持同步
  • 当 Vue 调用了 vm.$destroy ()，一般是 Vue 自己调用
  • beforeDestroy () => vm 中所有的 data，methods，指令等等，都处于可用状态，马上要执行销毁过程，一般再次执行：关闭定时器，取消订阅消息，解绑自定义事件等收尾操作
  • 拆卸所有的数据监测，子组件和自定义事件，原生的事件还能用
  • destroy () => 销毁，知道就行，吃灰函数

总结：

常用的生命周期钩子：

  • mounted：发送 ajax 请求，启动定时器，绑定自定义事件，订阅消息等初始化操作
  • beforeDestroy：消除定时器，解绑自定义事件，取消订阅信息等收尾操作

关于销毁 Vue 实例

  • 销毁后借助 Vue 开发者工具看不到任何信息
  • 销毁后自定义事件会失效，但原生 DOM 事件依然有效
  • 一般不会在 beforeDestroy 操作数据，因为即便操作数据，也不会在触发更新数据了
