先放启动流程接口调用链

以上的步骤完全可以在引擎中找到并对应起来。包括 IFileManager 的初始化，FTaskGraphInterface 的初始化，线程的开启，CoreUObject 模组的读取等等，概括了很多。

参考：https://www.cnblogs.com/kekec/p/15187684.html

以下是我在读 GEngine->Start () 后的一些发现和总结：

  1. 首先在 GEngine->Start () 之前，会调用 UGameInstance::CreateLocalPlayer 来创建本地玩家。如果是网络游戏，客户端的 LocalPlayer 会在连接上服务端的时候会在服务端进行注册，具体是在 GameMode::Login () 里面 RegisterPlayer ()，这个是在服务器完全启动之后的事情了。但还是交代一下。

  2. GameInstance->StartGameInstance () 里面处理的逻辑比较多，这里也只列出对应 API

    大致就是加载地图，初始化 Actor，调用 Actor 的 BeginPlay 函数，还有一些网络相关的处理，主要是为了理清引擎对这些 API 的调用顺序。对于新手来说可能会经常遇到，xxx is nullptr 的错误，其实有可能是因为没有初始化就获取而导致的。
