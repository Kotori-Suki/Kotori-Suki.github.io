# API 不正常工作 之 GetGameMode

前言：API 不正常工作大多原因都是咱自己用错了，或者不理解。一般不会是引擎的 bug

# Server 函数中调用 GetGameMode 返回空 (默认读者了解 RPC)

惯例先放 UGameplayStatic::GetGameMode 的实现

# 产生原因：

  • Server 函数定义在 Widget 等不可复制的类中
  • Server 函数定义在 Actor/Component 中，但这个 Actor/Component 不属于当前主机的 PlayerController

# 衍生问题：为什么不可复制的类里不能定义 RPC 函数？

  • RPC 函数跟其他函数一样，在游戏初始化的时候都会被编译成一个唯一标识符，也就是函数指针，在调用的时候实际上也只是用函数指针来找到这个函数，然后开辟一个栈空间，把参数压进栈中并执行函数。
  • ue 在初始化函数的时候，会根据反射来决定，是否要把这个函数定义为 RPC，但定义为 RPC 之前，引擎还回去检查这个类是不是可复制的，如果不是就不会编译为 RPC 函数。

# 衍生问题：为什么调用不属于 Controller 的 Actor/Component 中的 RPC 会不起作用？

  • RPC 的调用实际上也是依赖于 PlayerController 的，应该说服务端和客户端的所有数据交流都是依赖于 PlayerController 的。ue 的网络框架是 NetDriver + NetConnection + Channel + Actor/UObject 来实现的。而这个 NetConnection 就是一个 PlayerController 和服务器连接的通道，所有的数据会在不同的 Channel 里面传输，如 ControlChannel，VoiceChannel，ActorChannel。每个 Actor 都会有一个 Channel，如果这个 Actor 不属于 Controller，那就不会为这个 Actor 生成一个 ActorChannel，那当然这个 Actor 的 replication 和 RPC 都是不能生效的

# 解决办法：

  • 把这个函数定义到一个 Actor/UObject/Component 上 (UObject 可复制要重载函数)，然后把这个 Actor/UObject/Component 的 Owner 设置为 PlayerController
