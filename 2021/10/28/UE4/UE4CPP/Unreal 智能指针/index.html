# Unreal 智能指针

前言：记录一下研究 UE 智能指针的心得，然后简单实现一下。

# 概述：

总所周知，Unreal 的智能指针分三种：TSharedPtr，TSharedRef，TWeakPtr

使用方法可以到 unreal 的官方文档查，这里就不做过多赘述了。

如果学过 C++ 智能指针，相信对这仨指针理解起来也不难，道理是一样的，都是将普通指针和指针计数封装成类，然后重载类的方法，使这个类的使用上就跟普通指针没什么区别。

Unreal 实现智能指针的头文件有两个：SharedPointerInternal.h、SharedPointer.h

SharedPointerInternal.h 里面是智能指针的内部实现、SharedPointer.h 里面是智能指针的各种方法重载，以下我们将从这两个头文件入手，剖析 unreal 的智能指针。

# SharedPointerInternal.h

SharedPointerInternal.h 里面主要有五个类，分别是：FReferenceControllerBase，FRawPtrProxy，FReferenceControllerOps<ESPMode>，FSharedReference，FWeakReference

  • FReferenceControllerBase：引用控制器基类，里面记录了共享指针计数和弱指针计数，并提供 DestroyObject 抽象方法。所有智能指针内部都需要一个控制器基类对象。
  • FRawPtrProxy：原始指针代理结构体，包含一个原始指针和一个引用控制器，用于将原始指针转化为智能指针。
  • FReferenceCOntrollerOps<ESPMode>：里面是实现了智能指针的各种静态方法，比如获取指针计数，添加 / 释放指针引用等。
  • FSharedReference：共享引用类，主要是「构造函数」以及「等号运算符」的重载。初始化或改变类内部的原始指针的指向，改变引用控制器，并增加或减少引用控制器的计数等。
  • FWeakReference：弱引用类，主要也是「构造函数」以及「等号运算符」的重载，跟 FSharedReference 类似，不做赘述。

p.s. FSharedReference，FWeakReference 是最上层类，也是我们智能指针中真正使用到的类。其实质是一个引用控制器，和改变引用控制器的一些方法。

# SharedPointer.h

SharedPointerInternal.h 里面主要有四个类：TSharedRef，TSharedPtr，TWeakPtr，TSharedFromThis，以及 == , != , cast , MakeShareable 方法的实现与重载。

  • TSharedRef：俩变量，原始指针和 FSharedReference。方法主要也是「构造函数」以及「运算符」的重载以及一些成员函数。构造函数调用时会调用 FSharedReference 的构造函数，从而改变指针计数。等号运算符也是如此。需要注意的是，Init 函数是用于将原始指针转换为共享指针的方法，且该指针指向的对象必须继承于 TSharedFromThis，Init 函数是不支持继承于 TSharedFromThis 的 Object 指针转化为线程模式不同的 TSharedRef 的，Init 函数体里面的 EnableSharedFromThis 函数体用于初始化共享指针和对象指针的弱指针的。
  • TSharedPtr：基本和 TSharedRef 的实现相同，只是构造函数和等号运算符中，TSharedPtr 可以赋空值，TSharedRef 则不行。
  • TWeakPtr：俩变量，原始指针和 FWeakReference。方法主要是「构造函数」以及「运算符」的重载，还有 Pin 方法。Pin 方法是支持输出左值和右值的，函数体里判断该弱指针指向的 Object 是否仍被其他共享指针引用，如果有返回该共享指针，否者返回空的共享指针
  • TSharedFromThis：这个类并不是智能指针类，而是普通类的辅助类，继承于这个 TSharedFromThis 的类可以使用 AsShared 方法直接返回其共享引用，但前提是这个类必须被 TSharedPtr 初始化过，因为只有 TSharedPtr 初始化时才会初始化 TSharedFromThis 的 WeakThis 指针，否者 TSharedFromThis 对象将失去作用。
