# 2、先理清楚一些概念

关键词解析
OnlineSubsystem在线子系统，不同平台会有不同的在线子系统，如 "Steam“，但我们只需要用"NULL“ 子系统来进行开发，因为 NULL 子系统是所有其他子系统的基类，多态的好处相信都不陌生。在线子系统会提供给我们好友列表，唯一 ID，服务器管理器等，我们就能在网络上愉快地游戏了，而不仅限于局域网
WrapperFunctions包装函数，其实就是接口函数，直接用 Wrapper Functions 就可以兼容所有平台
Session会话，不是地图，也不是服务器，是一个概念性的名词。服务器可以在会话中创建，客户端可以加入到会话中。加入会话后会加载相应的地图。会话更像是数据库的入口，可以帮助我们跟踪所有正在运行的服务器
Session Interface这个跟 Session 是一样的，只是我们在开发的时候一般都会面向接口的开发，因为这不仅能让程序更健壮，扩展性和可复用性也都会更好
