# 进程 线程 多线程

概述：本文主要是对进程 线程 异步 多线程等概念的介绍，何时使用，以及注意事项等。

​ 进程和线程异步多线程其实都是操作系统中的概念，操作系统担任着计算机管理者的职位，负责任务的调度，资源的分配和管理，统领整个计算机硬件，包括 CPU。而应用程序实际上是被编译出来的可执行文件，是运行在操作系统之上的。

# 进程：

​ 进程是一个具有独立功能的程序在一个数据集上的一次动态执行的过程，是操作系统进行资源分配和调度的独立单位，是应用程序运行的载体。简单来说就是，操作系统会在你运行某个程序时，为这个程序分配一块内存，并将其可执行文件放在内存中，然后在 CPU 中运行，运行起来的程序就是一个进程。

进程具有的特征：

动态性：进程是程序的一次执行过程，是临时的，是动态产生，动态消亡的；

并发性：任何进程都可以同其他进程一起并发执行；

独立性：进程是系统进行资源分配和调度的独立单位；

结构性：进程由程序，数据，进程控制块三部分组成。

# 线程：

​ 线程是程序执行中单一的顺序控制流程，是程序执行流的最小单元，是处理器调度和分配的基本单位。一个进程可以包含一个或多个线程，各个线程之间共享进程的内存空间。（线程由线程 ID，当前指令指针 PC，寄存器和堆栈组成，而进程由内存空间和线程组成。）

# 进程和线程的区别：

  1. 线程是程序执行的最小单位，而进程是操作系统分配资源的最小单位；
  2. 一个进程有一个或多个线程组成，线程是进程中代码的单一执行路线；
  3. 进程之间相互独立，线程之间共享进程的内存；
  4. 线程切换比进程切换要快得多，因为切换线程只需要保存和设置少量寄存器的内容，而切换进程则需要保存进程 CPU 的环境和设置新调度进来的进程的 CPU 环境。简单来说就是进程的切换要保存和加载数据较大。
  5. 不同进程的线程进行切换会引起进程的切换。

# 线程调度算法

  1. 时间片轮转调度算法

    每个进程公平地，轮流地使用 CPU 一段时间，使用完后到队列后排队，在自己的时间片没执行完任务的线程会被强行剥夺 CPU 的使用权，会导致线程饥饿，且不区分任务优先级

  2. 优先级调度算法

  • 先来先服务调度算法（非抢占式）

  • 短作业优先调度算法（作业时间越短，优先级越高）

  • 优先级调度算法（自定义优先级）

  • 高相应比优先调度算法

    相应比 = (等待时间 + 服务时间) / 服务时间

  1. 多级反馈队列调度算法

维护一个队列数组，每个队列代表不同的优先级；

优先级高的队列中加入新线程会抢占优先级低的队列中线程的 CPU；

为每个线程划定一个执行时间片，执行超时则排到原队列队尾；

只有高优先级的队列中的线程都执行完了，低优先级的线程才能执行。

# 异步调度

前置知识：磁盘网络的读写操作一般都是直接内存访问的，是不需要消耗 CPU 时间的 I/O 操作。

异步调度就是为了让这些不消耗 CPU 时间的 I/O 操作异步执行，CPU 也就可以同时执行别的任务。等异步任务执行完了，会调用一个回调函数来处理特定的逻辑。异步唯一的难点就在于回调函数如何编写。

# 多线程

​ 与异步类似，多线程是指可以让主线程运行的同时去运行别的线程的操作。但不同于异步，多线程的每个线程都是需要 CPU 参与运行的。多线程对于多核 CPU 而言，就是把任务分配给不同的 CPU 来处理。而对于单核的 CPU 而言，也仅仅是让 CPU 在不同的时间片中执行不同的线程而已。而且线程切换的开销也是不小的，所以单核 CPU 中实现多线程，或者一次性创建过多的线程对性能的损耗还是挺大的。

# 死锁（线程同步）

​ 提到多线程就不得不提死锁的概念。学过操作系统的小伙伴应该都知道死锁的四个必要条件：

  1. 互斥条件：一个资源每次只能被一个线程使用
  2. 请求和保持条件：一个线程因请求资源而阻塞时，对已获得的资源保持不放
  3. 不剥夺条件：进程已经获得的资源，在未使用完之前，不能强行剥夺
  4. 循环等待条件：若干线程之间形成一种头尾相接的循环等待资源关系

简而言之就是，两个线程同时需要两个临界区的资源，每个线程各先获得其中一个，就会导致双方都没办法获取完全部需要的资源，双方都无法继续运行。（当然这里的线程数和所需的资源数都是可以大于 2 的）

# 处理死锁的策略：

  • 严格规范获取锁的顺序，即只有获得 A 锁后才能获得 B 锁
  • 开放调用，调用别的函数时不要加锁，或者说，加锁时不要调用其他函数。即如果你加了锁，你调用的那个函数又正好要这个临界区资源，就会造成死锁。
