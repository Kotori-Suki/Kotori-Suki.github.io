# 动态规划

概述：动态规划通常用在找最值问题上，按类别可以分为一维动态规划和二维动态规划，通常有两种办法求解，即迭代和递归，但相同点是都要找状态转移方程，把大问题转 s 换成小问题。

# 一维动态规划

** 经典问题：** 找零问题，即有 1、2、5 块钱的硬币来找零，怎么保证找回去的硬币最少

示例：

​ 输入：17
​ 输出：5
​ 解释：27 = 5 + 5 + 5 + 2 +1

解题步骤：

  1. 找状态转移方程

    状态转移方程：dp [amount] = min (dp [amount-1], dp [amount-2], dp [amount-5])+1;

    ​ 比如咱现在要找 28 块钱，那么找回去的硬币数就应该是在（找 27 块钱 + 1 个硬币，找 26 块钱 + 1 个硬币，找 23 块钱 + 1 个硬币）中返回使用硬币数最少的那个。对于（找 23 块钱 + 1 个硬币），同样可以重复上面的办法来再小化问题。

# 二维动态规划：

** 经典问题：** 最长重复子数组。给两个整数数组 A 和 B，返回两个数组中公共的、长度最长的子数组长度

示例：

​ 输入：
​ A: [1,2,3,2,1]
​ B: [3,2,1,4,7]
​ 输出：3
​ 解释：长度最长的公共子数组是 [3, 2, 1] 。

解题步骤：

  1. 找状态转移方程

    状态转移方程：

    ​ dp [a][b] = max (dp [a-1][b], dp [a][b-1]) // 两个数字不相同

    ​ dp [a][b] = max (dp [a-1][b], dp [a][b-1], dp [a-1][b-1]+1) // 两个数字相同

    解释：

    ​ 这里我们可以这么理解：给定 A 数组和一个空的 B 数组，再逐步增加 B 数组的元素。

    ​ 首先往 B 里面添加一个 3，很容易得出公共最长子数组为 [3]，为什么呢？看表格：

    空12321
    空000000
    3000111

    ​ 表格表明当前状态下的最大长度，这里我们把整个表格初始化为 0，如果有相同的 咱让它等于斜上角的数加一，如果不相同 咱就让它等于上面或者左边的数字中最大的一个。为什么这么规定呢？理解起来很简单，因为如果数字相同的时候咱要加一，咱就不能在这之前用过 A 中的这个 3，那么最后没用这个 3 且最大的数，那就是斜上角的数字了。数字不相同的时候呢，咱就按之前在 A 中或者在 B 中匹配到当前最大值更新即可。

    ​ 这样一来，我们只要把表格填完，答案 dp [5][5] 也就自然而然地出来了。表格如下：

    空12321
    空000000
    3000111
    2001122
    1011123
    4011123
    7011123

  2. 用代码把整个 dp [][] 画出来

结语：解决动态规划问题的方法就介绍到这里，剩下的就是去力扣刷题了哈哈，冲冲冲！！！
