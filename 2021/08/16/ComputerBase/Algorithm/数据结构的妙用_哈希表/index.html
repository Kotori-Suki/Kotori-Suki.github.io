# 数据结构的妙用 —— 哈希表

概述：本篇将总结哈希表的三种妙用

哈希表优势：可以记录数据，查找快，O (1)+O (n)

# 哈希集

例题 1：无重复字符的最长子串；子串要求是连续的；

如 "abcabcbb"，无重复字符的最长子串为 "abc" 或 "bca" 或”cab“，所以输出 3。

题解思路：

​ 首先这道题可能一开始会想用双指针来解决，但双指针每次都需要和前面所有未重复的字符比对，这会导致时间复杂度噌噌噌地就上到了 O (n2)，显然并不是最优解。那么最优解是怎么样地呢？那就是滑动窗口加哈希集。怎么个滑法呢？请看下图。

  • 绿色箭头表示循环的当前指针，对 string 进行一次遍历；
  • 绿色框框表示哈希集，keep 住当前没有重复地最长字串。

首先开始让绿色框框往右走直到碰到第一个重复字符 a，更新当前最大长度。然后让指针往后走，同时删掉绿色框框中的 a，然后发现框框又可以往右走了。重复这个步骤直到框框到达数组的最后，返回之前记录的最大值。

例题 2：两数之和，给定一个数组和一个值，返回数组中的两个数加起来等于这个值的下标。

如：输入：nums = [2,7,11,15]，target = 9
输出：[0,1]，因为 nums [0] + nums [1] == 9 。

​ 很简单的一道题，直接嵌套遍历就可以完成。但这么做时间复杂度会直接飙到 O (n2)，这是我们不愿意看到的。那么怎么办呢？当然是用空间换时间啦～（废话）

# 原地哈希

例题 3：缺失的第一个正整数。要求时间复杂度为 O (n)，空间复杂度为 O (1)

如：输入：nums = [3, 4, -1, 1]
输出：2，因为 [1, 3, 4, -1]，缺失的第一个正整数就是 2。

如：输入：nums = [3, 4, 2, 1]
输出：2，因为 [1, 2, 3, 4]，缺失的第一个正整数就是 5。

题解思路：

​ 很明显这题的难点在于时间和空间复杂度，如果我们直接建立一个哈希表，显然空间会超标，排序二分显然时间会超标。所以这里就要我们自己动手来用数组做一个哈希表啦哈哈

​ 因为正整数是从 1 开始的，所以这里的哈希函数就是把 i+1 映射到 nums [i] 中，那么如果 1 <= x <= nums.size()，x 一定能在 nums 中找到它映射的位置。如此一来，我们只需要把可以映射的 x 都映射到正确的位置，然后找出第一个映射不正确的下标返回来即可。
