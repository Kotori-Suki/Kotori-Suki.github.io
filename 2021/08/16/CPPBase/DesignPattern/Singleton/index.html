# 单例模式（Singleton）

概述：单例模式，即只有一个实例的类，该实例为静态实例，由其类在类内部创建。如：window 的任务管理器就是一个单例类对象。

# 特点：

  1. 单例类只有一个实例对象；
  2. 该实例对象由单例类自行创建；
  3. 对外提供一个访问单例对象的全局访问点

# 优缺点：

优点：

  • 单例类在内存中只有一个实例，减少内存开销；
  • 避免对资源的多重占用；
  • 设置全局访问点，优化和共享资源的访问

缺点:

  • 单例模式一般没有接口，难以扩展；扩展则需要修改原来的代码，不符合开闭原则；
  • 在并发测试中，单例模式不利于代码调试；在调试过程中，如果单例中的代码没执行完，也不能模拟生成一个新对象
  • 单例类的功能代码都写在一个类中，如果功能设计不合理，很容易违背单一职责原则。

# 应用场景：

  • 需要频繁实例化的类，使用单例可以降低系统的内存压力，减少 CG (C++ 中并没有，UE4 有)
  • 只能有一个对象的类，比如总统，每个人的身份证。
  • 单一控制逻辑的操作，比如任务管理器。
  • 对象需要被共享的场合，例如数据库的连接池。
  • 创建实例时占用资源较多，实例化耗时较长的，且经常使用的类
