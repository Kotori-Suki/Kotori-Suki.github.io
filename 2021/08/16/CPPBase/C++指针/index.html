# C++ 指针

概述：指针是 C++ 语言中特有的数据类型，不同于基本数据类型，指针是用来存放内存地址的，所以也是内存管理中很重要的一部分。本文将总结指针的基本用法和注意事项，以及二级指针的使用。

# 基本用法

  2. 声明一个指针，指向一个数据或对象（取值寻址）

# 注意事项

  1. 内存泄露问题，原因是：指针变量是存在栈内存中的，而用指针 new 出来的 对象 / 数据 是在堆内存中的。栈中指针会在当前程序块结束的时候自动释放掉，在此之前如果没有把该指针指向的堆内存释放掉，或者没有让别的指针指向这块堆内存，那就会再也没有办法访问这块堆内存，于是就造成了内存泄露。

  2. 野指针，原因：

    • 使用未初始化的指针，未初始化的指针存放的地址是随机的。

    • 某个指针指向的内存被释放掉了，而这个指针又没有被指向 nullptr，它就会成为野指针。

      指针指向的内存被释放掉的原因有很多，比如：栈内存在程序块运行结束时会自动释放，而指针的指向不会被重置为 nullptr。再比如：两个指针同时指向一块堆内存，其中一个释放掉了这块内存，另一个就会变成野指针，这里需要程序员手动把两个指针都重置为 nullptr。

  3. 内存碎片，原因：频繁地 new/delete 很小的内存，会导致堆内存被碎片化，最后可能导致申请不到内存的问题。这也和内存管理算法的实现相关，可以参考这篇文章：C++ 内存管理
