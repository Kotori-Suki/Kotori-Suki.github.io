# C++ 内存管理

概述：主要介绍 C++ 中的内存管理机制，两大段 (数据段、代码段)，五大区的分布和作用，说说我们的一些操作都是怎么申请到内存的，最后在来整理一下内存碎片产生的原因。

# 内存管理机制

C++ 程序会在汇编过程中从把操作系统分过来的内存分成两大段，即数据段和代码段。代码段存的是二级制代码，函数被编译后就是存在这里的。详细点会再被划分为五大区，即 TEXT (代码段)，GVar (全局变量，静态变量，常量)，BSS (未初始化的全局变量)，HEAP (堆)，STACK (栈)

BBS，GVAR，TEXT 没什么好说的，就是里面的数据只有在程序结束运行时才会被释放。

# 堆栈内存管理（重要）

回想一下，平时我们会怎么创建一个变量或者对象。

接下来可以回去看看刚刚那个模型，不难发现堆区和栈区中间是有一个未使用的内存空间的。栈区添加数据时是从上往下添加的，而堆区添加数据时是从下往上添加的，他们共享中间这边内存。(p.s. 模型中上面为高地址，下面为低地址)

栈区通常储存的数据

  • 函数调用栈中的局部变量，结构体，对象等，如：递归函数的调用栈往往是比较多的。
  • 子程序块中的局部变量，通俗的说 就是一个大括号里的局部变量。（包括 main 函数）

堆区通常储存的数据

  • malloc/new 出来的变量，结构体，对象都是存在堆内存中的。

# 堆栈区别

管理方式：栈内存是系统自动分配回收的，堆内存则需要程序员手动申请释放。

空间大小：栈空间通常只有 1~2M，如果超出就会栈溢出错误。堆空间则最大可达到 4G 之大。

是否产生碎片：栈的分配和回收是严格后进先出的，不会产生内存碎片。而堆空间频繁 new /delete 不同 size 的空间则会产生碎片，产生碎片的原因会在后面进行叙述。

生长方向：在内存中添加数据的方向，栈是从上往下的，而堆是从下往上的。

** 分配方式：** 栈内存可以由编译器静态分配或者动态分配，而堆内存只能动态分配。

** 分配效率：** 栈是机器系统提供的数据结构，计算机底层堆栈提供支持：分配专门的寄存器放栈的地址，压栈出栈都有专门的指令执行。而堆则是 C/C++ 库函数的算法实现内存分配的，算法的执行效率自然是没有计算机底层指令来的高效的。

# 注意：

  1. 无论是对栈的操作还是对堆的操作都一定不能越界，否则会导致程序崩溃或者其他意想不到的结果。
  2. 指针的堆栈操作一定要注意野指针和内存泄露问题，也就是堆栈的管理方式造成的问题。
  3. 栈中不会存放静态变量，因为那是存在静态数据区的，全局唯一。

# 产生内存碎片的原因

以下是个人总结，建议去看看这篇文章，里面说得很清楚

标记分配方式：

  1. 堆内存申请的方式是，从下往上的，且可以随时 delete。
  2. 如果我们频繁的 new 出不同的内存，然后又频繁释放掉其中某些内存，那就会导致堆内存碎片化，最后将导致无法申请到一块较大的内存。
  3. 例如，咱现在有 1G 的堆内存，然后咱 new 出 1024 块 1M 的堆内存，之后只释放掉偶数的堆，那么剩下的内存就全部是不连续的，碎片化的堆内存，如果这个时候我们需要申请一个 2M 的堆内存，那就会失败，因为没有连续的内存可以达到 2M，但我们明明还有 512M 的内存没有使用。

基于页面的内存管理方式：

  1. 维护一个用以查询内存 ID 是否被使用的页面（例如：每两个内存单位组成的一个内存空间）
  2. 在需要使用的时候只需再这个页面里面将空闲状态去除即可，这样一来，任意 ID 的内存空间就都是连续的了，跟链表的原理类似，在分页 ID 结束处指向另一个 ID 的开始处，也就是将物理内存转变成虚拟内存的过程。
  3. 如此，我们释放掉的内存就可以简单地用来申请更大的内存空间。
  4. 但这还是会导致内存碎片。因为我们是每两个内存单位组成一个内存空间的，如果我们每次申请内存都只需要一个内存单位，但我们一次最少申请一个内存空间，也就是两个内存单位。这样一来，将导致每次都浪费一半的内存空间（这样的情况会比较少，因为每次申请的空间一般都是成百上千的，所以还是有利于避免内存碎片的产生的）

p.s. 主流的内存管理方式都是基于页面的。
