# map 与 unordered_map

概述：map 和 unordered_map 都是字典结构，都是通过键值对来存取数据的。但他们几乎不是一个东西.... 面试被问到了.... 写篇文章来总结一下他们的区别。

# 区别：

  1. 引用的头文件不同

  2. 底层实现的数据结构不同

    • map：基于红黑树实现的树形结构。不存在扩容，红黑树节点都是满的
    • unordered_map：基于哈希表实现的数组。扩容：8 -> 64 -> 512 -> 1024 -> 2048 -> 4096 -> .......

  3. 元素排列顺序不同

  • map：红黑树的有序存储，本质上还是红黑树的性质
  • unordered_map：散列表的无序存储，本质上还是散列表的性质
  1. 插入和查询的时间复杂度不同
  • map：红黑树的插入查询复杂度，即 O (logn)
  • unordered_map：散列表的复杂度依赖于哈希冲突的多少，通常情况下是 O (1)

  1. 效率和稳定性不同

    • map 是红黑树，内存效率接近 100%，unordered_map 是哈希表，存在未使用的散列空间
    • 稳定性：map 是红黑树结构，数据量的增加对其查找插入操作的影响都较小，复杂度为 O (logn)。而哈希表结构的 unordered_map 的插入查询效率跟哈希冲突有关，最坏情况下的复杂度为 O (n)，即退化为数组。

# 优缺点：

  1. map：
    • 优点：
      • map 元素有序（这是 map 最大的优点，其元素的有序性在很多应用中都会简化很多的操作）；
      • 其红黑树的结构使得 map 的很多操作都可在 **O (logn)** 下完成；
      • map 的各项性能较为稳定，与元素插入顺序无关；
      • map 支持范围查找。
    • 缺点：
      • 节点占用的空间大：红黑树的每一个节点需要保存其父节点位置、孩子节点位置及红 / 黑性质，因此每一个节点占用空间大。
      • 查询插入操作的平均时间复杂度不如 unordered_map。
    • 适用场景：
      • 元素需要有序；
      • 对于单次查询时间较为敏感，必须保持查询性能的稳定性，比如实时应用等等。
  2. unordered_map
    • 优点：
      • 查询速度快，平均性能接近于常数时间 O (1)；
    • 缺点：
      • 元素无序；
      • unordered_map 相对于 map 空间占用更大，且其内存利用率不高；
      • 查询性能不太稳定，最坏时间复杂度可达到 O (n)。
    • 适用场景：
      • 要求查找速率快，且对单次查询性能要求不敏感。

总结：**map 是红黑树实现的，unordered_map 是哈希表实现的。** 其区别很大程度上就是红黑树和哈希表的区别。使用上如果需要元素有序或者对单词查询性能要求较为敏感时用 map，其他都可以用 unordered_map。
