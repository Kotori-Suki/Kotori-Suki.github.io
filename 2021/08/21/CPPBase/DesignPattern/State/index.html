# 状态模式（State)

概述：状态模式和策略模式如出一辙，就是表达的意思不一样。简单来说就是，状态模式表示一个物体的状态的变化，这是可以是不由用户控制的，但策略模式一般都是指由用户控制其变化的。

# 特点：

  1. 由一个状态抽象接口，多个状态实现类，和一个环境类组成，
  2. 环境类，需要有一个状态抽象接口的动态实例
  3. 改变不同状态时，只需改变环境类中的状态接口实例即可。

# 优缺点：

优点：

  • 不同状态下选择不同行为，即使状态较多也不会乱（相比 if-else）
  • 可以使用状态的抽象接口来实现抽象层的逻辑，符合依赖倒置原则
  • 增加状态也不需要修改原有代码，符合开闭原则

缺点:

  • 会增加类的个数，状态类过多时，难以管理
  • 不同状态的实现类都只能实现接口中定义的方法

# 应用场景：

  • 人物状态机的切换，状态模式可以设计得非常复杂且合理 :-）
