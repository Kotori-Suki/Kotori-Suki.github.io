# 装饰者模式（Decorator）

概述：装饰者模式，即用来装饰原型的改造体，原型是被装饰类，改造体即是我们要的装饰者类。

# 特点：

  1. 由一个抽象接口和一个被装饰者具体类，一个装饰者具体类组成
  2. 装饰者和被装饰者都继承自同一接口
  3. 装饰者可以扩展被装饰者的接口功能的同时，额外提供别的方法

# 优缺点：

优点：

  • 可以在不额外创建该实现类的派生类的前提下，拓展其功能，将新功能与原类解耦，符合开闭原则
  • 可以通过不同装饰类实现对原型的多种改造和变换，再通过组合不同装饰器获得具体的功能
  • 可以根据原型的不同来选用不同的装饰器，装饰器可以重用

缺点:

  • 装饰器过多不易管理
  • 每个装饰器都必须实现抽象接口的所有功能，所以接口隔离要做好

# 应用场景：

  • 需要扩展一个现有类的功能，且这个类不能或不应该生成太多子类时；
  • 想重用多种功能，且功能间常常需要不同的搭配组合时；
