# 策略模式（Strategy)

概述：即根据客户端的不同需求，选择不同的策略。比如：活动界面的切换，客户端切换活动界面来查看不同的活动，实际上对于任务栏来说，只是切换了不同的显示策略而已。再说简单一点就是，拥有一个抽象接口的升级版 if-else 语句，这个抽象接口的实现决定了 if-else 程序体的执行内容。

# 特点：

  1. 由一个策略抽象接口，多个策略实现类，和一个实用类组成，
  2. 使用类，需要有一个策略抽象接口的动态实例
  3. 需要使用不同策略时，只需改变使用类中的策略接口实例即可。

# 优缺点：

优点：

  • 不同状态下选择不同策略，即使策略较多也不会乱（相比 if-else）
  • 可以使用策略的抽象接口来实现抽象层的逻辑，符合依赖倒置原则
  • 增加策略也不需要修改原有代码，符合开闭原则

缺点:

  • 策略类过多时，难以管理（可以用哈希表存起来）
  • 不同策略的实现类都只能实现接口中定义的方法

# 应用场景：

  • 任务栏的任务页面切换，活动栏的活动页面切换等
  • LOL 中已配置好的符文页的切换
  • 每个策略的逻辑相似，但可以用来应对不同情况的时候
