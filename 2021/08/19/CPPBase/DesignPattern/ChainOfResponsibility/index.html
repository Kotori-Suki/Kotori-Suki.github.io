# 责任链模式（ChainOfResponsibility)

概述：责任链，即把存有解决方案的类连接成一个单向链表，遍历链表直至最后解决某个问题

# 特点：

  1. 有一个抽象节点类，里面定义了一个 next 节点，和责任链的纯虚方法
  2. 实现类需要实现抽象类的责任链方法，next 节点可以静态设置，也可以动态加载
  3. 处理某个责任时，我们只需要调用头节点的责任链方法即可。

# 优缺点：

优点：

  • 只需保存头节点，即可递归调用所有其他节点动态处理任务。
  • 需要增加责任节点非常简单，动态加载的节点可以删除，静态则需要修改原有代码
  • 扩展责任链方法无需修改原有代码，符合开闭原则

缺点:

  • 每个新生成一个责任处理方法都需要增加一个责任链节点类
  • 如果责任链过长，每次解决问题都需要遍历一遍链表，性能上会有损耗

# 应用场景：

  • 多个对象可以处理一类请求，但具体哪个对象处理该请求，在运行时动态决定
  • 对于出现的问题无法确认是哪一块的问题时，可用责任链动态处理
