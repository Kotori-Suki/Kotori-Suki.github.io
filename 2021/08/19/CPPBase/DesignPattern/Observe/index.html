# 观察者模式（Observer)

概述：观察者模式，即有若干观察者他们都在等待一个被观察者的信号，获取到这个信号后执行相应的行为

# 特点：

  1. 由一个观察者抽象接口，若干具体观察者和一个被观察者抽象类，一个被观察者具体类组成
  2. 观察者需要在被观察者抽象接口中注册信息
  3. 被观察者抽象类中需要维护一个列表储存所有注册过的观察者
  4. 某状态成立时，需要被观察者遍历整个观察者列表中的抽象方法

# 优缺点：

优点：

  • 不需要每个观察者都时时刻刻都去关心被观察者的情况，提高系统性能
  • 将观察者和被观察者解耦，不需要在观察者里面访问被观察者。

缺点:

  • 一个被观察者抽象类只能处理一种回调函数，处理多种回调函数会导致接口增多
  • 如果观察者把抽象方法注册到多个被观察者上时，是谁调用的这个方法将不得而知

# 应用场景：

  • 一个对象的状态变化，会对多个对象产生影响时
  • 当抽象模型有两方面以上，且一方面的变化依赖于另一方面时，如：MVC
  • 实现简易的多播委托
