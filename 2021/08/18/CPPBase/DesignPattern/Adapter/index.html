# 适配器模式（Adapter）

概述：适配器模式，即将两个本来接口不兼容的类适配到一起，从而实现对不兼容接口的调用。按类型可以分为类适配器和对象适配器。

# 特点：

  1. 由一个抽象适配接口，若干个具体适配器，和若干个被适配者组成
  2. 通常一个适配器对应一个适配者，客户端通过适配器接口调用被适配者的方法

# 优缺点：

优点：

  • 客户端可以直接调用抽象层的适配器接口，而不需要了解具体如何适配的
  • 不需要因为接口不同而重写代码，复用了原来的代码
  • 可以解耦被适配者和客户端的代码，符合开闭原则

缺点:

  • 用不上的地方就不用啊.... 感觉没什么缺点（

# 应用场景：

  • 现存的满足需求的类使用的接口和目标系统接口不匹配
  • 使用第三方组件时，接口定义可能不一致

区别：

  • 类适配器就是继承适配接口的同时，还继承了需要适配的类，然后直接在适配器内调用被适配者的函数来实现的；
  • 对象适配器则是将被适配者实例化后，用其对象调用被适配者函数的方式来实现的；
  • 所以对象适配器占用的内存是会比类适配器大的

结语：适配器模式常常会在接口不兼容时用到，用不到就先想想别的模式嘛哈哈
