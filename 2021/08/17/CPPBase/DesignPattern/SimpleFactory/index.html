# 简单工厂模式

概述：简单工厂模式，即一个工厂只负责生产一类产品（继承于同一个抽象接口的）

# 特点：

  1. 由一个简单工厂，一个抽象产品，若干具体产品组成；
  2. 简单工厂类负责创建所有具体产品实例；
  3. 简单工厂创建实例的方法通常都是静态方法

# 优缺点：

优点：

  • 客户端创建具体产品时不需要知道类名，只需要知道参数即可。
  • 可以引入配置文件，在不修改客户端代码的情况下更换和添加新的具体产品类

缺点:

  • 简单工厂模式的工厂类单一，负责所有产品的创建，职责过重，违背单一职责原则
  • 简单工厂类代码非常臃肿，违背高聚合原则
  • 扩展困难，一旦增加新产品不得不修改工厂逻辑，不符合开闭原则
  • 具体产品较多时，会造成逻辑过于复杂
  • 简单工厂模式使用 static 工厂方法，造成工厂角色无法形成继承的等级结构

# 应用场景：

  • 产品种类相对较少时，可以考虑使用简单工厂模式。
