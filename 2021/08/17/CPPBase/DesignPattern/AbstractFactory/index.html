# 抽象工厂模式（AbstractFactory）

概述：上一篇说过，工厂模式是为每一件产品单独做一个工厂，这样虽然解耦了，也维护了抽象层。但还是有个很致命的缺点，就是具体产品类过多时将会导致工厂类也非常之多。那么有没有办法让工厂保持原来的优势的同时，生产更多的产品呢？答案就是：抽象工厂模式

# 特点：

  1. 由一个或多个工厂抽象接口，多个产品抽象接口，若干具体工厂，若干具体产品组成
  2. 每个具体工厂负责生产一个或多个具体产品

# 优缺点：

优点：

  • 可以在类的内部对产品族中相关联的多等级产品共同管理；
  • 增强了程序的可扩展性，增加新产品时，不需要修改源代码，符合开闭原则

缺点:

  • 增加新产品时需要对所有工厂类进行修改；
  • 增加了系统的抽象性和理解难度

# 应用场景：

  • 当需要创建的对象时一系列相互关联或相互依赖的产品时，如锁和钥匙
  • 系统中有多个产品族，但每此都只使用其中的某一产品，如苹果的 Iphone
  • 系统中提供产品的类库，且所有产品接口相同，客户端不依赖产品实例的创建细节和内部结构

    相信大家也看出来了，这里的抽象工厂模式是要求每个工厂都要具备生产原本接口所规定的产品的能力的，比如说接口中规定工厂要有生产手机和电脑的能力，那么我们实现出来的工厂就必须都满足。但这会导致另外一个问题，那我要是这个品牌他只生产手机，不生产电脑呢？那这里我们的抽象工厂显然就是违背了接口隔离原则的，要解决这个其实也不难，就是把抽象工厂的接口分成多个即可，然后再用某个产品的工厂接口来获得特定的产品

    • 以下是将抽象工厂的不同产品生产方法拆开的代码：
    //抽象产品手机
class AbstractMobile {
public:
	virtual void display() = 0;
};

//抽象产品电脑
class AbstractComputer {
public:
	virtual void display() = 0;
};

//抽象工厂手机
class AbstractFactoryForMobile {
public:
	virtual AbstractMobile* GetMobile() = 0;
};

//抽象工厂电脑
class AbstractFactoryForComputer {
public:
	virtual AbstractComputer* GetComputer() = 0;
};

//具体手机Iphone
class IPhone : public AbstractMobile {
public:
	void display() {
		cout << "This is a IPhone" << endl;
	}
};

//具体电脑MacBook
class MacBook : public AbstractComputer {
public:
	void display() {
		cout << "This is a MacBook" << endl;
	}
};

//具体工厂AppleFactory，这里这个工厂既可以生产手机，也可以生产电脑，所以继承两个接口
class AppleFactory : public AbstractFactoryForMobile, public AbstractFactoryForComputer {
public:
	AbstractMobile* GetMobile() {
		return (AbstractMobile*) new IPhone();
	}
	AbstractComputer* GetComputer() {
		return (AbstractComputer*) new MacBook();
	}
};

//具体电脑Alien
class AlienBook : public AbstractComputer {
public:
	void display() {
		cout << "This is a AlienBook" << endl;
	}
};

//具体工厂AlienFactory，这里这个工厂只生产电脑，所以继承只继承 AbstractFactoryForComputer
class AlienFactory : public AbstractFactoryForComputer {
public:
	AbstractComputer* GetComputer() {
		return (AbstractComputer*) new AlienBook();
	}
};


int main() {
	AbstractFactoryForMobile* MobileFactory;
	AbstractFactoryForComputer* ComputerFactory;
	AbstractMobile* Mobile;
	AbstractComputer* Computer;
    //AppleFactory开始生产手机和电脑
	MobileFactory = new AppleFactory();
    ComputerFactory = new AppleFactory();
	Mobile = MobileFactory->GetMobile();
    Computer = ComputerFactory->GetComputer();
	Mobile->display();
    Computer->display();
    //AlienFactory开始生产电脑
	ComputerFactory = new AlienFactory();
	Computer = ComputerFactory->GetComputer();
	Computer->display();
}

//运行结果:
//This is a IPhone
//This is a MacBook
//This is a AlienBook
